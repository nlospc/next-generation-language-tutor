Known issues

  • No lint script exists yet.
  • CLI install remains a placeholder from the skeleton phase.
  • The TypeScript source tree is still a skeleton and there is no TypeScript build script yet.
  • OpenAI-compatible provider is implemented but still not wired to gateway (by phase design).
  • MCP server is a minimal stdio JSON-RPC implementation for Phase 4 and may need protocol hardening in later phases.
  • Web UI uses server-rendered HTML and minimal in-page JavaScript only; no frontend framework is used yet.
  • Phase 7 rewrite currently uses deterministic mock LLM provider output by default; OpenAI-compatible rewrite exists but is not default wired for regeneration.
  • Phase 9 implements Telegram command adaptation locally but does not yet run a live Telegram Bot API polling or webhook transport.
  • Phase 10 training generation is deterministic local prompt/task generation; it does not yet call a live LLM to render full stories, dialogues, or conversation replies.
  • No active Python source exists in this repository; only generated Python cache files were seen locally and are now ignored.
  • Native module compatibility can block tests: if better-sqlite3 was built against a different Node ABI, run npm rebuild better-sqlite3 before interpreting test failures.
  • MCP tool argument quality depends on inputSchema richness; scripts/mcp-server.mjs now exposes explicit argument schemas for all core tools, not only add_learning_item.
