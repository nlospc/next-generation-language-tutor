# Agent Instructions

You are developing an agent-facing application that turns an installed agent into a language tutor.

Project requirements are in `docs/`. Read the relevant documents before planning or coding. Treat `docs/` as the source of truth. Do not invent product scope.

Work incrementally. Before each change, inspect the existing code and choose the smallest safe next step. Reuse existing files, patterns, and utilities where possible. Do not rewrite working code unless required.

Keep the implementation restrained. Avoid unnecessary abstractions, dependencies, frameworks, generated boilerplate, or speculative features. Prefer simple, direct, maintainable code.

Maintain `docs/development-progress.html` throughout development. Update it after meaningful changes so another agent can resume without hidden context. It should record: current status, completed work, files changed, open questions, known issues, and next step. Use simple browser-readable HTML only.

If requirements are unclear, incomplete, conflicting, or require a product decision, stop and ask the user before coding. Do not guess on important behavior.

After changes, run the relevant checks available in the repository, such as build, test, or lint. Record what was run and any failures in `docs/development-progress.html`.
